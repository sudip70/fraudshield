System Architecture
How raw transaction input moves through feature engineering, model inference, decision logic, and impact analysis.
Pipeline Flow
1) Data + Feature Layer
Builds behavior, geo, temporal, and account-risk features (distance, unusual time, prior fraud, velocity, international mismatch).
2) Model Benchmarking
Compares LR, RF, XGBoost and other candidates with stratified CV; best model selected using imbalance-aware metrics.
3) Decision Layer
Combines ML probability with rule triggers to output practical actions: APPROVE, REVIEW, or BLOCK with full traceability.
4) Explainability
Uses SHAP (global + local) so each score is auditable and understandable for stakeholders and reviewers.
5) Business Impact Layer
Converts precision/recall trade-offs into operational cost, savings, and threshold-sensitive impact projections.
Feature Design Rationale
  • Behavioral drift features capture deviations from personal norms (frequency and amount spikes).
  • Geographic inconsistency (home vs transaction location) surfaces high-friction fraud patterns.
  • Merchant novelty and prior fraud history improve early-risk signal strength.
  • Temporal cues (late-night activity, weekend patterns) improve separability in edge cases.
  • Rules intentionally complement the model for high-severity scenarios where recall matters most.
How To Read This Dashboard
  • Exploratory: Understand fraud hotspots and pattern concentration.
  • Model Performance: Compare metrics, calibration, confusion matrix, and threshold tuning.
  • Live Scorer: Score one transaction with explanation, decision trace, and risk signals.
  • Business Impact: Translate model behavior into FP/FN costs and projected savings.
  • Goal: Prioritize explainable, decision-ready fraud triage over raw model accuracy alone.
Reference Papers
  • Davis & Goadrich (2006) — Why Precision-Recall curves are preferred for imbalanced classification.
  • Breiman (2001) — Random Forest methodology and variance reduction.
  • Chen & Guestrin (2016) — XGBoost system design for scalable gradient boosting.
  • Lundberg & Lee (2017) — SHAP unified framework for model explanations.
  • Niculescu-Mizil & Caruana (2005) — Probability calibration insights for supervised models.

Distance from Home
